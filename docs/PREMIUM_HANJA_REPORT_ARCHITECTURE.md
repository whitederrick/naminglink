# 프리미엄 한자 작명 리포트 구조

## 상품 원칙

- 2,900원 상품은 한자 후보 5개의 상세 설명과 한자 종합 상세를 제공한다.
- 4,900원 상품은 한자 후보 10개의 상세 설명과 한자 종합 상세를 제공한다.
- 9,900원 상품은 후보 10개와 한자 종합 상세, 사주·오행 분석 및 PDF를 제공한다.
- 결제 완료 시점부터 구매 결과를 24시간 동안 다시 확인할 수 있으며 PDF는 9,900원 상품에만 제공한다.
- 24시간이 지나면 출생 정보, 분석 결과와 PDF 파일을 삭제한다.
- 결제·환불 처리를 위한 최소 거래 기록은 관계 법령과 별도 보존 정책에 따라 분리 보관한다.
- 전통 명리 참고 분석을 제공하며 과학적 예측, 운명 또는 특정 오행 보완을 단정하지 않는다.
- 후보는 서비스에 반영된 공식 인명용 한자 자료를 기준으로 검토하되, 최종 등록 가능 여부는 신고 시점의 대법원 조회에서 다시 확인하도록 안내한다.

## 계산과 DB의 역할 분리

### 코드에 고정하는 계산

- 음력·양력 변환
- 절기 기준 사주 원국(년주·월주·일주·시주)
- 천간과 지지의 기본 오행 매핑
- 경도 기반 출생 시각 보정

기준 엔진은 `@fullstackfamily/manseryeok` 1.0.8이다. 같은 결제 결과를 다시 만들 수 있도록 엔진 버전을 세션에 기록한다. 계산 결과를 AI가 임의로 만들거나 수정하지 않는다.

### DB에서 버전 관리하는 자료

- KASI 음양력·24절기 교차검증 이력
- 전통 명리 해석 규칙과 검수 상태
- 인명용 한자의 오행 분류, 분류 근거와 검수 상태
- 해석 문구, 출처와 주의 문구
- 결제 세션, 24시간 만료 시각과 PDF 파일 메타데이터

`saju_reference_versions`가 production이 되려면 경계일을 포함한 KASI 표본 검증과 해석 규칙 검수가 먼저 끝나야 한다.

## 필수 입력

사주·오행이 포함된 9,900원 상품에서는 무료 화면의 두 시간 단위 선택만 사용하지 않는다. 다음 정보를 별도 결제 화면에서 정확히 받는다. 출생 연·월·일이 미정이면 2,900원과 4,900원 상품만 이용할 수 있다.

- 양력 또는 음력
- 음력인 경우 윤달 여부
- 출생 연·월·일
- 출생 시·분 또는 시각 모름
- 출생 지역 또는 경도
- 대한민국 서비스 기준 시간대 `Asia/Seoul`

출생 시각을 모르면 시주와 시간 보정을 제외하고 `PARTIAL_NO_TIME`으로 명시한다.

## 결제·생성 흐름

1. 서버가 결제 주문을 생성한다.
2. 결제사 웹훅의 서명과 주문에 저장된 상품별 금액(2,900원·4,900원·9,900원)을 서버에서 검증한다.
3. 중복 웹훅은 `payment_webhook_events`의 provider event ID로 차단한다.
4. 주문 생성 시 서버가 원본 접근 토큰을 한 번만 발급하고 SHA-256 해시만 DB에 보관한다. 원본 토큰은 결제 브라우저에만 저장하며 URL에는 넣지 않는다.
5. 계산 엔진이 구조화된 사주 결과를 만든다.
6. production 상태의 해석 규칙과 공식 한자 자료만 매칭한다.
7. 웹훅은 결제 상태만 반영하고 30초 안에 종료한다. 상세 해설은 결제 완료 후 별도 요청으로 공개하며 사주 해설과 PDF는 9,900원 상품에서만 생성한다.
8. PDF는 비공개 `premium-reports` 버킷에 저장한다.
9. 다운로드 요청마다 접근 토큰과 만료 시각을 검사한 뒤 최대 60초의 서명 URL을 발급한다.
10. 만료 정리 작업이 PDF와 분석 데이터를 삭제한다. 파일 삭제 실패 시 결과 데이터는 먼저 비우고 파일 삭제를 재시도한다.

## 포트원 V2 연결

- 결제 모듈: PortOne V2 카드 결제(`CARD`)
- 권장 채널: 토스페이먼츠 신규 모듈
- 결제 웹훅: `/api/payments/portone/webhook`
- 서버 검증: `PAID`, paymentId, 상점 ID, 주문에 저장된 상품별 금액, KRW를 모두 다시 조회한다.
- 멱등 처리: `webhook-id`와 원문 SHA-256을 `payment_webhook_events`에 저장한다.
- 모바일 리디렉션: 접근 토큰은 브라우저 localStorage에 보관하고 URL에는 세션 ID와 결제 ID만 사용한다.

필수 환경변수는 `NEXT_PUBLIC_PORTONE_STORE_ID`, `NEXT_PUBLIC_PORTONE_CHANNEL_KEY`, `PORTONE_API_SECRET`, `PORTONE_WEBHOOK_SECRET`이다.

로컬 개발에서는 결제 우회 버튼이 자동 표시되며 실제 사주 계산·OpenAI 해설·PDF 생성 경로를 사용한다. 운영에서 `PREMIUM_TEST_MODE=true`를 설정하면 우회 버튼이 노출되므로 내부 시험 후 즉시 제거해야 한다.

## 운영 전 남은 게이트

- KASI Open API 기준 경계일 회귀 테스트 작성 및 통과
- 사주 해석 규칙과 한자 오행 분류의 전문가 검수
- Supabase Cron 또는 Vercel Pro Cron에 hourly cleanup 스케줄 등록과 `CRON_SECRET` 설정
- 포트원 실제 가맹점 키로 카드 결제·취소 웹훅 통합 시험

Vercel Hobby Cron은 하루 한 번만 실행할 수 있어 정확한 24시간 삭제에 적합하지 않다. Supabase Cron에서 `/api/cron/premium-cleanup`을 매시간 호출하거나 Vercel Pro의 hourly Cron을 사용한다.

현재 PDF 샘플은 `pnpm premium:render-sample`로 생성하며 `tmp/pdfs/premium-hanja-report-sample.pdf`에 저장된다. 전체 CJK 글꼴 대신 공식 인명용 한자 9,493자와 사주 간지·복성에 필요한 글자만 포함한 OFL 글꼴을 사용한다.
