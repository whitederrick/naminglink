## 1. 프리미엄 PDF 마무리 (커밋 ebd28cc, 8118a99)

- **표지 로고**: 하단 워터마크 방식 폐기 → 우측 상단에 `position:absolute`(width 60, opacity 0.35)로 랜딩 로고 배치. 레이아웃 밀림 원천 차단. pymupdf PNG 렌더로 시각 확인.
- **푸터**: `NAMING-LINK(GLOBAL NAMING STUDIO) · PREMIUM NAMING REPORT` (fixed, 전 페이지).
- **종합 비교 해설 반복 문구 해결**: 원인은 ①폴백이 단일 템플릿, ②4개 섹션이 한 호출에 묶여 금지어 1건이면 전체 폴백되는 전부-아니면-전무 구조. → 비교 해설을 **전용 OpenAI 호출(후보별 items 배열)**로 분리, 항목 단위 검증(문제 항목만 폴백), 번호·"한자 —" 형식은 코드에서 부여, 모델이 붙인 접두어(번호·한자·조사)는 정규식으로 제거. 폴백도 후보별 자의·초점·마무리 문장이 달라지게 재작성.
- **PDF 파일명**: `naminglink-premium-{한글이름}.pdf` — signed URL(download 파라미터)과 운영자 테스트 다운로드(anchor.download) 양쪽 적용.
- 로컬 렌더 스크립트의 `@/lib` 별칭 문제는 `npx tsx --tsconfig tsconfig.json`으로 해결(스크립트는 프로젝트 루트에서 실행해야 폰트 경로가 맞음. pnpm 때문에 스크래치패드 밖 스크립트는 openai 패키지 못 찾음 → 임시 파일을 프로젝트 안에 두고 실행 후 삭제).

## 2. 한글→글로벌 이름 변환 품질 개편 (커밋 978c390, 9e22f39, e977cc9, 7ab55bf, 12e36b8)

### 진단(실호출 검증으로 확인된 문제)
사주 입력 완전 무시(원시 문자열만 전달), 지역·목적·톤이 상투구로만 소비, 일본어 대상 시 설명 전체가 일본어로 출력(outputLanguage 버그), 비실존 이름 조합(ナムタク 등), 의미 연결이 원 이름과 무관, 동음이의 한자 혼동(奎→規).

### 적용된 해결(우선순위: ①데이터 주입 ②서버 확정 교정 ③프롬프트)
- **입력 폼 세분화**: 성·이름·성 한자·이름 한자·이름 의미(textarea).
- **outputLanguage=ko 강제**(클라이언트+서버 이중).
- **사주 실계산 주입**: `buildKoreanToGlobalSajuReference`(openai.ts) — 만세력 엔진으로 오행 요약 생성.
- **공식 한자 DB 근거 주입**: `getOfficialHanjaMeanings`(official-hanja-db.ts) — 입력 한자의 지정 음가·뜻을 official_hanja_entries에서 조회해 `originalNameElements`로 전달. 근거 우선순위 절대 규칙: ①공식 조회 표 ②nameMeaning 원문 ③소리만.
- **서버 확정 교정** `normalizeKoreanToGlobalResult`(openai.ts): 로마자권 full_name_local("안 Nathan"→"Nathan Ahn", 여권식 성 로마자 표 60여 개), pronunciation("가스파르 / 가스파르"→"Gaspard / 가스파르"), 비로마자권 한글 성 잔재 제거(가타카나·키릴 성 표 30개 + 로마자 폴백).
- **프롬프트**: 5개 변환 전략(중복 금지), 실존 이름 강제 + few-shot 작업 예시(김도윤→悠真(ゆうま) 좋은/나쁜 예), name_meaning(어원 이야기 2~3문장), meaning_connection은 원 이름 요소 기준, matching_rate 채점 기준(소리30+의미30+현지25+목적15, 약한 후보 85 상한), 성 발음 규칙(안→アン·Ahn·Ан), 언어별 name 문자 규칙은 사용자 메시지 필드(`nameScriptRule`)로 주입.
- temperature 0.6(이 서비스만).
- **22개 국가/언어 전수 검사**: 자동 검증 스크립트(문자 체계·한글 혼입·발음 형식·설명 언어·후보 수)로 22/22 통과. 스크립트: 스크래치패드 `k2g-sweep.ts`(회귀 검사 재사용 가능).
- **UI**: 결과 카드 전용 레이아웃(현지 표기, 성까지 붙인 전체 표기, 변환 전략, 이름의 의미, 현지 인식), 일본어 제목 陽翔(ようすけ) 형식, "매칭률 N%"→"적합도(참고) N점".

### 잔여 한계
변환 전략 라벨 정확성(소리 보존안 오라벨·중복)은 실행마다 흔들림 — gpt-4o로만 해결되는 영역. **gpt-4o 실측: 품질 명확히 우위, 20.7초, ~28원/건. 업그레이드는 보류 중**(적용 시 openai 클라이언트 timeout 30s와 /api/naming maxDuration 조정 필요).
